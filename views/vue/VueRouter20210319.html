    # 什么是前端渲染和后端渲染

    后端渲染： 类似于php与jsp，前后端未分离，jsp内包含html+css+java传递给服务器，可以说在服务器内已经存放好了一个页面.

    后端路由： 在后端处理URL和页面之间映射的关系。 每个页面有自己对应的网址 URL会发送到服务器，服务器通过正则进行URL匹配，最后交给Controller进行处理 Controller进行各种处理，最终生成HTML或者数据，返回给前端 完成一个IO操作

    前端路由： 简单来说就是一个按钮会生成一个url一个url映射我们的vue组件 称作为SPA页面，单页面应用，只有一个index.html页面，通过前端路由映射页面 可以查看网易云音乐网页版，全程不存在url的跳转 前端管理映射关系，在vue中就是通过vue-router进行管理

    # 改变URL但页面不刷新

    • 通过url的hash location.hash = ''
    • HTML5的history.pushstate({},","home") 这个可以返回
      • history.back() 返回上一个目录
      • history.go(-1) 同上
      • history.replacestate({},","home") 这个不可返回

    # router-link属性补充

    tag：tag可以指定<router-link>之后渲染成什么组件 例如tag="button" 但是在router4中这个属性被移出了

    replace:让返回键失效，直接加上即可

    active-class：每个link点击时都会添加上一个默认的router-link-active类 可以通过这个属性来修改点击时添加上的类名

    也可以在配置文件中修改点击时添加的默认类名

    const router = new VueRouter({
	routes,
	mode:'history',
	linkActiveClass:'active'
})

    # 路由懒加载

    问题：打包构建应用时，JS包会非常大，一次性请求影响页面加载，例如所有组件被打包到一个js中

    解决：将路由对应的组件打包成一个个的js代码块，用到时再加载对应组件

    使用：

    使用路由懒加载写法

    const routes = [
    {
		path:'/home',
		component:() => import('../components/Home')
	},
	{
		path:'/about',
		component:() => import('../components/About')
	},
]

//另一种写法，之前的下面不变上面的import改成
const Home = () => import('../components/Home')
const About = () => import('../components/About')

    # 路由的嵌套

    需求由来：许多时候一个组件中还要嵌套其余组件，例如home中有news的按钮，通过/home/news访问

    实现步骤：1.创建对应的子组件，在路由映射中配置对应的子路由 2.在组件内部使用<route-view>

    举例实现：1.创建了两个子组件，一个HomeNews一个HomeMessage，分别对应不同的内容 2.在组件的配置文件中给home配置children 3.在home组件中使用这两个子组件和子路由

    const News = () => import('../components/HomeNews')
const Message = () => import('../components/HomeMessage')
//同样使用懒加载

{
		path:'/home',
		component:Home,
		children:[
			{
				path:'',                //这里作为重定向
				redirect:'news'
			},
			{
				path:'news',  //子路由不需要加上/默认会在之前的/home后面加上
				component:News
			},
			{
				path:'message',
				component:Message
			}
		]
	}

    # 路由参数传递

    • params方式

    路由配置形式：/router/:id

    传递方式：在path后面跟上对应的值

    传递后形成的路径：/router/123

    例子：见上面的动态路由

    • query方式

    路由配置形式：/router

    传递方式：对象中使用query的key作为传递方式

    传递后形成的路径：/router?id=123

    例子：

    <router-link :to="{path:'/user',query:{
                  name='hello',age=12}}">

//切记传递对象一定要用v-bind    
//在user.vue中获取

{{$route.query.name}}
{{$route.query.age}}

    # 路由守卫

    用于监听路由之间的跳转从而实现一些方法，例如修改文档的title

    {
    path：'user',
    component:User,
    meta:{
        title:'用户页'
    }
}

//router的配置文件中

router.beforeEach((to,from,next)=>{
    //从from跳转到to
    document.title=to.matched[0].meta.title   //给每个路由配置meta对象
    next()
})

    # keep-alive

    keep-alive可以用于避免组件再切换时被摧毁，同时可以使用生命周期中的activated和deactivated来控制两个状态

    //注意要添加<keep-alive></keep-alive>
activated(){
    this.$router.push(this.path).catch(err => err);
}
beforeRouteLeave(to,from,next){
   this.path = this.$route.path;
   next()
}
