    VueX

    婶铛铛 2021/3/20 Vue

    # 解释

    Vuex是一个状态管理工具

    简单的看成把需要多个组件共享的变量全部存储在一个对象里面，这个对象放在顶层的Vue实例中

    # 好处

    Vuex带来的遍历就是响应式的，共享的对象可以响应处理

    # 共享的对象

    用户登录状态、用户名称、头像、地理位置信息

    商品的收藏、购物车的物品

    # 安装

    npm install vuex --save

    # 使用步骤

    • 安装完成vuex之后，创建store文件夹，再在文件夹内创建index.js
    • 在index.js内配置完以下内容
    • 在main.js中import store from './store/index.js'
    • 在new Vue中使用store
    • 完成以上步骤即可使用vuex

    # 过程图

    # state

    这个对象存放的是全组件共享的数据

    state单一状态树：整个vue实例中只使用一个store，不要创建多个store

    • 使用： $store.state.name即可使用其中的name数据
    • 修改：在没有异步操作下的修改使用到mutation

    # mutation

    用于对state数据的修改

    • 创建：在mutation中创建方法，记住要传入state

    • 使用： 在vue实例中创建方法来使用mutation，两个rename同名但不相同

      ​ 要用到this.$store.commit(mutation)

      ​

      ​

    • 传递参数-Payload： 只需要在this.$store.commit(mutation,参数) 并且在mutation中接收参数

    • 提交风格： 另一种提交形式，这样的形式可以提交更多的参数

    • 响应式原理：提前在store中初始化好所需的属性

      ​ 在state中的对象添加新属性或删除属性是，要用Vue.set、Vue.delete ​ Vue.set(state.user,'age',15)

    • 推荐： 官方推荐将所有的mutation在另一个文件中定义为常量

    # getter

    偏向于计算属性，就是先将state中的数据处理了再使用

    • 创建： 直接在getters内创建方法即可，要传入state

    • 使用： 直接使用$store.getter.twoname

    • 补充：getters创建的方法可以有第二个参数getters，及使用上面的getters进一步修改 getters也可以传参，但是需要return出一个函数

    基本类似于对mutation进行异步处理，是一个中间商，传递的参数是context

    # modules

    对store进行模块划分，解决state单一树问题

    • 模块的state：$store.state.a.name moduleA
    • 模块的mutation：与原来的写法不变，mutation命名不重复
    • 模块的getters：与原来相同，但如果要用到根模块state，用rootState
    • 模块的actions：里面的context对应的是模块里的上下文

    # 文件目录

    官方推荐将mutation、getters等都抽离出去生成新的js文件，利用import导入，其中state的内容可以放在index.js
