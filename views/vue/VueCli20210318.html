    Vue Cli

    婶铛铛 2021/3/18 Vue

    # 说明

    vue-cli可以快速搭建Vue开发环境以及对应的webpack配置 称作脚手架

    # 使用前提

    Node和Npm和webpack

    # 安装

    npm install -g @vue/cli

    由于现在以及更新到了cli4，所以要通过命令来拉取cli2或者cli3的模板，这样就可以创建不同版本的脚手架

    # 初始化项目

    • vue-cli2:vue init webpack my-project
    • vue-cli3:vue create my-project

    # Cli2创建过程

    1. Project name: 项目名称 保存在package.json 直接回车与文件夹名字相同

    2. Project description： 项目表述 test vue cli2

    3. Author： 作者 xxx 邮箱

    4. 选择模式: 分别是runtime-compiler与runtime-only

      默认选择的是上面的模式，更保险

      但其实第二个模式的运行效率高且源代码量更少

    5. vue-router： 是否安装

    6. ES-Lint: 用于检查js的代码规范 容易疯狂报错

    7. unit tests： 单元测试 一般不用

    8. e2e tests: 自动测试 一般不用

    9. 管理工具: 有npm/yarm

    # 关于runtime模式的区别

    # runtime-compiler

    template -> ast -> render -> vdom ->UI

    # runtime-only

    render -> vdom ->UI

    因此下面这种模式性能更高，且代码量更少，但需要用到render函数

    # 说明：

    runtime-only 其实只能识别render函数，不能识别template，vue文件中的template也是被 vue-template-compiler 翻译成了render函数，所以只能在.vue里写 template

    # 总结

    • 如果在之后的开发中，依然使用template，就需要选择Runtime-Compiler
    • 如果在之后的开发中，使用的是.vue文件夹开发，那么可以选择Runtime-only

    # Cli3创建过程

    1. 选择配置 默认or手动
    2. 选择手动的话，会弹出许多配置让自己选择
    3. 选择配置文件的位置 单独 放在package.json
    4. 是否保存自己的配置

    # Cli3补充

    开启图形化界面：vue ui

    自定义配置： 通过创建vue.config.js来新增自己想添加的配置文件
