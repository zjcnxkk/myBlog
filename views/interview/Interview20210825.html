    # 1.Babel的原理

    说明

    js在不断的发展，但是由于浏览器的多样性，需要将我们的代码让低版本的浏览器也能兼容。Babel就是一个工具，把用最新标准编写的 JavaScript 代码向下编译成可以在今天随处可用的版本

    原理

    babel的转移过程分成三个阶段：

    1. 解析Parse：将代码解析生成抽象语法树(AST)，即词法分析和语法分析的过程。
    2. 转换Transform：通过AST一系列的变换，babel接受得到AST通过babel-traverse对其进行遍历，在此过程中进行添加、更新及移除等操作。
    3. 生成Generate：将变化后的AST再转化为JS代码，通过使用babel-generator

    # 2.如何写一个Babel的插件

    # 3.git的工作流

    GitFlow是一个git操作流程标准，包含如下几个关键分支：

    • master：主分支
    • develop：主开发分支 包含确认即将发布的代码
    • feature：新功能分支 一般一个新功能对应一个分支，对于代码的拆分需要比较合理，避免冲突
    • release：发布分支 发布时候用的分支
    • bug：bug分支 再这个分支进行修复hotfix分支，紧急修bug的时候用

    GitFlow的优势：

    1. 并行开发：每个新功能都会建立一个新的feature分支，从而和已经完成的功能隔离开来，再新功能开发完成的情况下，合并到主开发分支develop
    2. 协作开发：每个feature分支上改动的代码都只是为了让某个新的feature可以独立运行，同时我们很容易知道每个人在干什么
    3. 发布阶段：develop分支主要用来暂时保存那些还没有发布的内容，可以包含好多feature
    4. 支持紧急修复：hotfix分支是从某个已经发布的tag上创建出来并做一个紧急的修复，这个紧急修复只影响这个已经发布的tag

    # 4.rebase与merge的区别

    git rebase和gir merge一样都是用于从一个分支获取并且合并到当前分支

    假设一个场景，就是我们开发的feature/todo分支要合并到master主分支，那么用rebase或者merge有什么不同呢？

    marge特点：自动创建一个心动commit 以免合并时遇到冲突可以修改
优点：记录了真实的commit情况，包括每个分支的详情
缺点：因为每次merge会自动产生一个merge commit，所有看到分支会杂乱

    rebase特点：会合并之前的commit历史
优点：得到更简洁的项目历史，去掉了merge commit
缺点：合并出现的代码问题不容易定位，因为rewrite了history

    因此需要保留详细的合并信息的时候建议使用git merge,特别是需要将分支合并进入master分支时：当发现自己修改某个功能时，频繁进行git commit提交时，发现其实过多的提交信息没有必要时，可以尝试 git rebase

    # 5.git reset 、git revert和git checkout有什么区别

    git仓库有三个组成部分：工作区、暂存区、历史记录区

    • 工作区：在git管理下的正常目录都算是工作区，我们平时的编辑工作都是在工作区完成
    • 暂存区：临时区域，里面存放将要提交文件的快照
    • 历史记录区：git commit 后的记录区

    git reset 、git revert和git checkout的共同点：用来撤销代码仓库中的某些更改

    不同点：

    从commit层面：

    • git reset可以将一个分支的末端指向之前的一个commit，然后再下次git执行垃圾回收的时候，会把这个commit之后的commit都扔掉。git reset还支持三种标记，用来标记reset指令影响的范围：

      --mixed：默认选项，会影响暂存区和历史记录区
--soft：只影响历史记录区
--hard：影响工作区、暂存区和历史记录区
      1
      2
      3

    • git checkou可以将HEAD移到一个新的分支，并更新工作目录 执行这个指令之前， 你需要 stash 或者 commit 暂存区和工作区的更改

    • git revert 和git reset目的相同，但做法不同他会以创建新的commit的方式来撤销commit，这样能保留之前的commit历史，比较安全。 执行这个指令之前， 你需要 stash 或者 commit 暂存区和工作区的更改

    从文件层面：

    • git reset 只是把文件从历史记录区拿到暂存区，不影响工作区的内容，而 且不支持 --mixed、--soft和--hard
    • git checkout 则是把文件从历史记录拿到工作区，不影响暂存区的内容
    • git revert不支持文件层面的操作

    # 6.webpack和gulp的区别(模块化与流的区别)

    gulp

    强调前端开发的工作流程，我们可以通过一系列的task，定义task处理的事务（例如文件压缩合并、雪碧图、启动server、版本控制等），然后定义执行顺序，让gulp执行这些task，从而构建整个前端开发流程

    webpack

    webpack是一个前端模块化方案，侧重模块打包，我们可以把开发中的所有资源都看成模块，通过loader和plugins对资源进行处理，打包成符合生产环境部署的前端资源

    # 7.谈谈你对Vue的理解

    Vue是一个构建数据驱动的渐进性框架，它的目标是通过API实现响应数据绑定和视图更新。

    # 8.Vue的优缺点

    优点：

    1. 数据驱动视图，对真实dom进行抽象出virtual dom，并配合diff算法，响应式、观察者和异步队列等手段用最小的代价更新dom且渲染页面
    2. 组件化，组件用单文件的形式进行代码的组织编写，使得我们可以在一个文件里编写html、css、js并配合Vue-loader之后支持更强大的预处理器等功能。
    3. 强大且丰富的API提供一系列的api能满足业务开发的各类需求
    4. 虚拟dom让Vue ssr先天就足
    5. 生命周期钩子函数，选项式的代码组织方式，写熟悉了还是蛮顺畅的，且仍有优化空间
    6. 生态好社区活跃

    缺点：

    • 底层基于Object.defineProperty实现响应式，而整个api本身不支持IE8及以下的浏览器
    • csr的先天不足，首屏会有性能问题
    • 引擎爬虫无法爬取js中的内容，故spa先天就对seo优化心有余而力不足。

    # 9.什么是虚拟dom

    虚拟dom是相对于浏览器所渲染出来真实dom的，在react、vue出现之前，我们要改变页面展示的内容只能通过遍历查询dom树的方式，来找到需要修改的dom，这种方式相当消耗计算资源，因此用建立的虚拟dom对象，也就是js对象，用对象嵌套的方式来表示dom，那么每次dom树修改就变成了js对象的修改，这样性能开销小。

    # 10.描述以下vue的生命周期

    生命周期就是 vue 从开始创建到销毁的过程，分为四大步（创建，挂载， 更新，销毁）

    • beforeCreate前也就是nue Vue的时候会初始化事件和生命周期；
    • beforCreate和create之间会挂载Data，绑定事件；
    • 根据el挂载页面元素，如果没有设置el则生命周期结束，直到手动挂载；
    • el挂载结束后，根据templete/el渲染页面；
    • 在beforeMount前虚拟DOM已经创建完毕；
    • 在mount前，将vm.$el 替换掉页面元素 el；
    • mount将虚拟dom挂载到真实页面；
    • 之后数据变化触发beforeUpdate、updated进行一些操作；
    • 最后主动调用销毁函数或者组件自动销毁时beforeDestory，手动撤销监听事件、计时器等
    • destory时仅存在Dom节点，其他所有东西已自动销毁。

    # 13.删除数组用delete和Vue.delete有什么区别

    • delete：只是被删除数组成员变成empty/undefined,其他元素键值不变
    • Vue.delete：直接删了数组成员，并改变了数组的键值

    # 14.watch和计算属性有什么区别

    computed： 用来声明式的描述一个值依赖了其他的值，当所依赖的值或者遍历改变时，计算属性也会跟着改变；

    watch： 监听的是已经在data中定义的变量，当该变量变化时会触发watch的方法

    但是既能用watch又能用computed实现的功能，推荐computed，原因在于其有缓存功能。

    # 15.Vue双向绑定原理

    Vue的数据双向绑定是通过数据劫持结合发布者-订阅者模式的方式来实现的。

    利用Object.defineProperty()这个方法重新定义了对象获取属性值(get)和设置属性值(set)

    # 16.v-model是什么，有什么用

    v-model用于双向绑定，相当于 v-bind:value="xxx" 和 @input，意思是绑定了一个value属性的值，子组件可以对value属性监听，通过$emit('input', xxx)的方式给父组件通讯。

    # 17.axios是什么，怎么使用，怎么解决跨域

    axios是一种异步请求，用法与ajax类似。

    通过npm安装即可使用，包括get、post、put、patch、delete等五种请求方式。

    解决跨域问题可以在请求头添加Access-Control-Allow-Origin

    也可以在index.js文件中更改proxyTable配置等解决跨域问题

    # 20.Vue3.0为什么要用Proxy API替代defineProperty API

    主要是为了响应式优化

    • .defineProperty API 的局限性最大原因是它只能针对单例属性做监听
    • Proxy API 的监听是针对一个对象的，那么对这个对象的所有操作会进入监听操作，这 就完全可以代理所有属性，将会带来很大的性能提升和更优的代码
    • 响应式是惰性的，执行 Object.defineProperty 把每一层对象数据都变成响应式的，这 无疑会有很大的性能消耗
    • Proxy API递归响应式，真正访问到的内部属性才会变成响应式，简单的可以说是按需实现响应式，减少性能消耗

    # 21.Vue3.0编译做了哪些优化

    • Block tree

      在2.0里，渲染效率的快慢与组件大小成正相关：组件越大，渲染效率越慢。并且，对于一些静态节点，又无数据更新，这些遍历都是性能浪费。

Block tree是一个将模版基于动态节点指令切割的嵌套区块，每个区块内部的节点结构是固定的，每个区块只需要追踪自身包含的动态节点。所以，在 3.0 里，渲染效率不再与模板大小成正相关，而是与模板中动态节点的数量成正相关
      1
      2
      3

    • slot编译优化

      Vue.js 2.x 中，如果有一个组件传入了 slot，那么每次父组件更新的时候，会强制使子组件 update，造成性能的浪费。

Vue.js 3.0 优化了 slot 的生成，使得非动态 slot 中属性的更新只会触发子组件的更新。动态 slot 指的是在 slot 上面使用 v-if，v-for，动态 slot 名字等会导致 slot 产生运行时动态变化但是又无法被子组件 track

    • diff算法优化

    # 22.Vue3.0的composition API

    Vue Composition API 围绕一个新的组件选项 setup 而创建。setup() 为 Vue 组件提供了 状态、计算值、watcher 和生命周期钩子。 并没有让原来的 API（Options-based API）消失。允许开发者 结合使用新旧两种Api。

    # 23.Vue3.0如何变得更快

    • diff方法优化：新增静态标记，在虚拟dom对比上更快
    • hoistStatic 静态提升：对不参与更新元素进行不停的复用，不再重新创建
    • cacheHandlers 事件侦听器缓存：对事件进行缓存复用

    # 25.vue在created和mounted这周期中请求数据的区别

    一般请求数据在created里面就行，如果涉及到需要页面加载完成之后的话就用mounted。

    在created的时候，视图中的html并没有渲染出来，所有此时去操作html的dom节点，找不到相关元素。

    而mounted中，已经渲染出来，可以直接操作dom节点。

    # 26.说说你对Proxy的理解

    vue的数据劫持有两个缺点

    1. 无法监听通过索引修改数组的值的变化
    2. 无法监听object也就是对象的值的变化

    因此vue2.x才推出$set属性的存在

    proxy式es6中推出的新的api，可以弥补以上两个缺点。

    vue3.0版本也用proxy替代了object.definproperty
