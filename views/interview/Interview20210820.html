    # 1.说一下CSS盒子模型

    css中的盒子模型包括标准的W3C盒子模型和IE盒子模型 **W3C盒子模型： ** 从里到外就是content-box、padding-box、border-box。这里的width和hight表示的是content部分

    IE盒子模型： 与W3C的相同，主要是width表示的是content+padding+border 这三个部分的宽度

    那么计算盒子宽度的时候，W3C需要左右 border+左右 padding+width，而IE只需要width

    CSS3盒子模型： CSS3引入box-sizing属性，其中box-sizing:content-box表示标准的盒子模型，box-sizing:border-box表示的是ie盒子模型。另外box-sizing:padding-box表示padding+content

    # 2.画一条0.5px的线

    • 利用meta viewport的方式，缩放0.5倍

    • 利用transform：scale（）这个是css中的缩放功能

      transform: scaleY(0.5);

    # 3.link标签和import标签的区别

    • link属于html的标签，而import是css提供
    • 页面加载时link会被同时加载，而@import引用的css文件会在页面加载结束后加载
    • link没有兼容性的要求，@import要ie5以上才能识别
    • link方式样式的权重高于@import

    # 4.transition和animation的区别

    两者大部分的属性相同，都随时间改变元素的属性值。

    但transition需要触发事件才行，而animation不需要触发任何事件就可以。

    transition为两帧，从from...to

    而animation可以一帧一帧

    # 6.BFC

    BFC(Block formatting context)直译为"块级格式化上下文"。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

    如何创建BFC

    • float的值不是none
    • position的值不是static、relative
    • display的值：inline-block、table-cell、flex、table-caption、inline-flex
    • overflow的值不是visible

    BFC的作用

    • 利用BFC避免margin重叠
    • 自适应两栏布局
    • 清楚浮动

    # 8.JS动画与CSS3动画差异

    渲染线程分为main thread和compositor thread

    如果css动画只改变transform和opacity，那么这时整个css动画得以在compositor thread，而js动画则会在main thread，然后出发compositor thread进行下一步操作，特别注意的是如果改变 transform和 opacity 是不会 layout 或者 paint的

    两者区别

    • js功能覆盖面比css大
    • 实现难度不同，css3比js更加简单，性能跳优方向固定
    • 对帧数表现不好的低版本浏览器，css3可以做到自然降级
    • css动画有天然事件支持
    • css3有兼容性问题

    # 9.说一下块元素和行元素

    块元素

    独占一行，会自动填满父元素，可以设置margin、padding以及高度和宽度

    行元素

    不会独占一行，width和height失效，并且垂直方向的padding和margin失效

    # 11.三种隐藏的区别

    opacity=0：元素隐藏，但不会改变页面布局，仍能触发事件

    visibility=hidden：元素隐藏，但不改变页面布局，不能触发事件

    display=none：元素隐藏，会改变页面布局，理解为元素被删掉

    # 12.外边距折叠

    多个相邻（兄弟或者父子关系）普通流的块元素垂直方向margin会重叠

    重叠结果

    • 两个相邻的外边距都是正数时，折叠结果是它们两者之间较大的值。
    • 两个相邻的外边距都是负数时，折叠结果是两者绝对值的较大值。
    • 两个外边距一正一负时，折叠结果是两者的相加的和。

    # 13.Position属性比较

    fixed

    固定定位，元素的位置相对于浏览器窗口是固定位置，即使窗口滚动它也不会移动。位置与文档流无关，不占据空间，和其他元素重叠

    relative

    相对定位，相对于”起点“进行移动，元素仍占据原来的空间，会覆盖其他框

    absolute

    绝对定位，该定位的元素位置相对于最近已定位的父元素，若没父元素，则相对于html。位置与文档流无关，不占据空间，也会和其他元素重叠

    sticky

    粘性定位，元素先按照普通的文档流定位，然后相对于该元素在流中的flow root（BFC）和containing block（最近的块级祖先元素）定位。元素定位表现为在跨越特定阈值前为相对定 位，之后为固定定位。

    static

    默认值，没有定位，出现在正常的流中

    inherit

    继承父元素的定位

    # 16.CSS的选择器和优先级

    • 选择器有：id选择器、class选择器、标签选择器、伪元素选择器、伪类选择器
    • 优先级顺序为：id 选择器 > class 选择器 > 标签选择器
    • 伪类选择器和属性选择器优先级相同
    • 继承祖先元素的样式的优先级比较低
    • 带有!important 标记的样式属性的优先级最高
    • 样式表的来源不同时，优先级顺序为：内联样式> 内部样式 > 外部样式 > 浏览器用户 自定义样式 > 浏览器默认样式

    # 17.CSS的动画如何实现

    创建动画需要使用到CSS3的animation属性或其子属性，该属性允许配置动画时间、时长以及其他动画细节，再通过@keyframes来配置动画的实际表现，来定义动画序列小节部分。

    transition也可以实现动画，但这个强调过度，是元素的一个或多个属性发生变化时产生的过渡效果。

    第二个途径当某种改变发生 （例如 hover）时才能获取样式，这样就会产生过渡动画。

    # 19.三栏布局的实现方式

    两列定宽，一列自适应

    • float+margin
    • float+overflow 给 div 设置 float：left，再给 right 的 div 设置 overflow:hidden。这样子两个盒子浮动，另 一个盒子触发 bfc 达到自适
    • 利用position 父级 div 设置 position：relative，三个子级 div 设置 position：absolute，这个要计算好盒 子的宽度和间隔去设置位置，兼容性比较好
    • table 父级 div 设置 display：table，设置 border-spacing：10px//设置间距，取值随意,子级 div 设置 display:table-cell，这种方法兼容性好，适用于高度宽度未知的情况，但是 margin 失效，设计间隔比较麻烦
    • flex实现
    • grid parent的div设置 display：grid，设置 grid-template-columns属性，固定第一列第二列宽度，第三列auto

    # 25.inline-block、inline、block

    block：块级元素，前后都有换行符，能设置宽高、边距

    inline：行内元素，宽高无效，margin竖直方向无效，padding有效，前后无换行符

    inline-block：能设置宽高，间距在垂直方向上都有效，但是前后无换行符

    # 26.重绘和重拍，如何减少

    重排

    DOM 的变化影响到了预算元素的几何属性比如宽高，浏览器重新计算元素的几何属性，其他元素的几何属性也会受到影响，浏览器需要重新构造渲染书。

    重绘

    浏览器将收到影响的部分重新绘制到屏幕上

    注意点

    重排一定导致重绘，重绘不一定导致重排

    产生原因

    • 添加或删除DOM元素
    • 元素尺寸位置的改变
    • 浏览器页面初始化
    • 浏览器窗口大小的改变

    解决方法

    1. 不在布局信息改变时做 DOM 查询
    2. 使用 csstext,className一次性改变属性
    3. 使用 fragment 对于多次重排的元素，比如说动画。
    4. 使用绝对定位脱离文档流，使其不影响其他元素

    # 27.overflow的原理

    块格式化上下文：CSS 可视化渲染的一部分，它是一块区域，规定了内部块盒 的渲染方式，以及浮动相 互之间的影响关系

    那么当设置overflow且值部位为visible时，这个元素就构建了一个BFC在计算高度时，内部浮动元素的高度也会计算在内，从而达到了清除浮动的目的。
